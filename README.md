Порівняльний аналіз швидкодії доступу до БД (Node.js)
Даний проєкт реалізований на платформі Node.js для оцінки ефективності обробки запитів до СУБД MySQL. Дослідження порівнює два підходи: прямі запити (Raw SQL) та використання ORM-технології.
1. Структура проєкту (Node.js)

  /speedRaw — модуль, що використовує драйвер mysql2 для виконання прямих SQL-запитів.
  
  /speedOrm — модуль, реалізований за допомогою ORM-бібліотеки Sequelize.
  
  transportcomppany.sql — файл для ініціалізації структури бази даних (3НФ).


2. Підготовка до запуску
  База даних:
  
    *Встановіть MySQL Server.
    
    *Імпортуйте структуру таблиць з файлу transportcomppany.sql.
  
  Середовище:

   *Переконайтеся, що встановлено Node.js (версія 16+).  
   *В обох папках (speedRaw та speedOrm) виконайте команду для встановлення залежностей:
   
    npm install

3. Запуск тестування
  Для отримання результатів виконайте наступні команди:
  
--Тест прямих запитів:

    cd speedRaw
    
   Модуль реалізований за допомогою бібліотеки mysql2, яка є низькорівневим драйвером для виконання прямих SQL-запитів та забезпечення максимальної швидкодії взаємодії з БД
    Запуск:
    
    node speedRaw.js
    
  Робота з меню - при запуску з'явиться меню === Database Performance Test Menu ===:  
    Пункт 1 (SELECT test):  
    *вимагає введення LIMIT (напр. 100) та автоматично тестує всі сутності предметної області (клієнти, водії, замовлення тощо).  
    Пункт 2 (JOIN test):   
    *дозволяє обрати тип об'єднання (INNER, LEFT, RIGHT) та виміряти швидкість складного запиту по 6 таблицях одночасно.  
    Пункт 3 (INSERT -> UPDATE -> DELETE):  
    *проводить циклічне тестування модифікації даних (рекомендовано для таблиці order  
  
--Тест ORM (Sequelize):  

    cd speedOrm
    
   Робота з модулем ORM (speedOrm)  
    Модуль реалізований за допомогою бібліотеки Sequelize, що дозволяє працювати з БД через об'єктну модель.  
    Запуск:   
    
    node speedOrm.js
    
  Інструкція користувача: При запуску з'явиться меню === Database Performance Test Menu (ORM Only) ===:  
    Пункт 1 (SELECT test):  
    *Програма запитає: Enter LIMIT for SELECT (ORM).  
    *Введіть значення (наприклад, 100).  
    -Результат виводиться у вигляді таблиці з середнім (Avg), мінімальним (Min) та максимальним (Max) часом у мілісекундах   
    Пункт 2 (INSERT -> UPDATE -> DELETE):  
    *Програма запитає: Enter table name for INSERT/UPDATE/DELETE (ORM).  
    *Введіть: order або іншу таблицю на ваш розсуд.  
    -Програма запустить цикл зі 100 операцій вставки, оновлення та видалення об'єктів.
